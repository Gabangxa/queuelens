The problem

3 dashboards for 3 frameworks

  • ✗BullMQ? Open Taskforce.sh at $95/mo — or self-host Bull Board inside your app code.
  • ✗Sidekiq? SSH into your Rails server and open the embedded web UI.
  • ✗Celery? Run flower locally and hope your Redis is accessible.
  • ✗Customer notices the failure before you do — because there are no cross-framework alerts.
The solution

1 dashboard. No code changes.

  • ✓Paste a Redis URL → QueueLens auto-discovers all queues. No app changes required.
  • ✓All frameworks in one view: BullMQ, Sidekiq, and Celery side by side.
  • ✓Click any failed job to see the full payload, error message, and stack trace.
  • ✓Retry or discard jobs directly from the UI — no Redis CLI needed.
